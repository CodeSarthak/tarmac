# Tarmac

Pre-flight cost estimation CLI for Claude Code.

## Build & Run

```bash
npm install
npm run build        # compiles src/ → dist/
tarmac-cost setup     # installs hooks into ~/.claude/settings.json
tarmac-cost estimate  # called automatically by Claude Code hook (reads stdin)
tarmac-cost report    # called automatically on session end (reads stdin)
```

## Architecture

- `src/cli.ts` — CLI entry point (commander)
- `src/commands/` — setup, estimate, report commands
- `src/core/conformal-predictor.ts` — trained regression model + conformal prediction intervals
- `src/core/formatter.ts` — formats cost estimate for Claude to present
- `src/core/prompt-classifier.ts` — classifies prompt into task types
- `src/core/context-estimator.ts` — estimates prior context from transcript
- `src/core/token-counter.ts` — token counting (API or heuristic)
- `src/data/model-weights.ts` — trained model weights (auto-generated by train-model.ts)
- `src/data/pricing.ts` — Claude model pricing

## Training & Validation

- `train-model.ts` — trains the regression model on SWE-bench data
- `validate-conformal.ts` — head-to-head validation vs old heuristic
- `signal-analysis.ts` — feature importance analysis

## Key Design Decisions

- Conformal prediction for calibrated coverage guarantees (80% target → 81% actual)
- Per-model regression (separate weights for Opus/Sonnet/Haiku)
- Zero runtime ML dependencies — just feature extraction + dot product
- Hook-based: intercepts prompts via Claude Code's UserPromptSubmit hook
